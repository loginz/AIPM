# 第一阶段交付物定义（AI Agent V1）

本文档用于定义智能物业报告项目第一阶段的交付边界、系统形态、功能范围、部署方式和使用流程，确保项目启动时目标清晰、范围可控、便于实施。

---

## 1. 文档目的

第一阶段的核心目标不是建设一个完整的智能物业管理平台，而是先交付一个**可落地、可试用、可验证价值**的最小版本。

这个最小版本的本质是：

**一个面向物业报告自动生成的 AI Agent 工作流系统。**

它不是单纯的一个聊天机器人，也不是完整 ERP/物业管理系统，而是一个围绕“资料采集 -> 信息抽取 -> 报告初稿生成”构建的半自动化 Agent 应用。

---

## 2. 第一阶段交付物的定位

### 2.1 它是不是 AI Agent？

是，但不是“只有一个 Agent”。

更准确地说，第一阶段交付的是：

- 一个 AI Agent
- 一套标准模板与字段规则
- 一套原始资料归档规范
- 一套人工审核发布流程
- 一套最小可运行的本地或内部部署工具

因此，第一阶段的交付物应该被定义为：

**物业报告生成 Agent V1**

### 2.2 它的角色是什么？

这个 Agent 的角色不是替代物业经理，而是作为“报告准备助手”。

它主要负责：

- 读取资料
- 理解资料
- 抽取关键信息
- 组织章节内容
- 生成报告初稿

物业经理仍然负责：

- 审核关键数据
- 增加管理判断
- 修正不准确表述
- 批准正式版本

---

## 3. 第一阶段的交付目标

第一阶段目标建议定义为：

### 目标 1：打通从原始资料到报告初稿的流程

系统能够从指定目录中读取物业资料，并生成一份可供人工审核的报告初稿。

### 目标 2：验证 AI 是否能节省实际工作时间

重点不是追求“全自动”，而是验证：

- 是否能减少资料整理时间
- 是否能减少报告撰写时间
- 是否能减少遗漏关键事项的风险

### 目标 3：建立后续系统化建设的基础资产

第一阶段应沉淀出后续可复用的：

- 报告模板
- 字段字典
- 归档规则
- AI 提示词规则
- 结构化输出格式

---

## 4. 第一阶段系统定义

### 4.1 系统名称建议

建议命名为：

**物业报告生成 Agent V1**

或内部代号：

**Property Report Agent V1**

### 4.2 系统本质

它是一个“面向任务的 AI Agent”系统，而不是传统意义上的单页面业务系统。

它的核心能力包括：

- 文件扫描
- 文件分类
- 内容抽取
- 数据归类
- 结构化输出
- 文本生成

### 4.3 系统运行模式

建议第一阶段采用：

**人工触发 + AI 执行 + 人工审核**

而不是：

- 实时自动执行
- 无人审核自动发报
- 大范围自动决策

---

## 5. 第一阶段必须交付的内容

第一阶段建议至少交付以下 6 项内容。

### 5.1 AI Agent 执行模块

负责：

- 扫描指定目录
- 识别输入文件类型
- 调用 AI 对文件进行抽取与归类
- 输出结构化结果
- 按模板生成 Markdown 报告初稿

### 5.2 报告模板

基于今天已完成的模板文档，固定第一阶段使用的输出结构。

建议至少包含章节：

- 项目概况
- 财务摘要
- 欠费与催缴
- 合同与执照
- 维修与运营事项
- 安保与现场事件
- 公用事业
- 待决事项

### 5.3 字段字典

定义 AI 需要抽取的字段，例如：

- report_month
- property_name
- management_fund_income
- arrears_total_amount
- contracts_expiring_90_days
- major_incidents

### 5.4 原始资料归档规范

明确资料如何按物业和月份归档，方便 Agent 自动读取。

### 5.5 审核与发布流程

明确哪些内容必须由人工确认，例如：

- 财务金额
- 欠费名单
- 法律/投诉措辞
- 需委员会审批事项

### 5.6 示例输入与示例输出

每个试点物业建议至少准备：

- 一套样本资料
- 一份人工原报告
- 一份 AI 自动生成初稿

用于对比效果和验证准确率。

---

## 6. 第一阶段输入范围

为保证范围可控，第一阶段建议只处理以下三类输入。

### 6.1 Excel

包括：

- 财务报表
- 欠费表
- utility 表
- 合同/执照台账

### 6.2 PDF

包括：

- 月报
- 会议资料
- 安保报告
- 财务说明
- 合同/证书扫描件

### 6.3 图片

包括：

- 巡检照片
- 维修照片
- 现场异常照片
- 文档拍照

### 第一阶段暂不优先接入

- 实时聊天系统
- 邮件自动实时抓取
- 实时工单系统对接
- 自动审批流
- 自动对外发报

这些可放到第二阶段或第三阶段。

---

## 7. 第一阶段输出范围

第一阶段的输出建议明确为两层。

### 7.1 结构化输出

AI Agent 应先输出一份结构化结果，例如 JSON、CSV 或数据库记录。

内容包括：

- 财务数据
- 欠费数据
- 到期合同
- 到期执照
- 工单和事件摘要
- 水电数据
- 待决事项

### 7.2 报告初稿输出

在结构化结果基础上，再生成：

- Markdown 月报初稿

后续如需要可再导出为：

- Word
- PDF

---

## 8. 第一阶段不做的内容

为了控制范围，建议明确以下事项**不属于第一阶段交付范围**。

### 不做 1：完整物业管理系统

第一阶段不是要交付：

- 收费系统
- 工单系统
- CRM 系统
- 发票审批系统

### 不做 2：自动决策

Agent 不直接替代管理处做以下决定：

- 法律追讨决定
- 合同续约决定
- 投诉结论裁定
- 财务审批

### 不做 3：实时监听所有渠道

第一阶段先不做：

- 实时监控聊天群
- 自动抓邮箱全部内容
- 实时监听共享盘变化后秒级处理

### 不做 4：完全无人审核出报告

最终报告仍需人工审核。

---

## 9. 第一阶段推荐部署形态

第一阶段建议采用轻量部署，不追求复杂架构。

### 9.1 推荐形态

可采用以下组合：

- 一个共享资料目录
- 一个 Agent 配置文件
- 一个本地或内网运行的 Agent 程序
- 一个结构化输出目录
- 一个报告输出目录

### 9.2 推荐目录结构

```text
项目根目录/
  原始资料/
    [物业名称]/
      [月份]/
        01_财务/
        02_欠费/
        03_合同与执照/
        04_维修与工单/
        05_安保/
        06_公用事业/
        07_图片/
  中间结果/
  输出报告/
  配置/
```

### 9.3 推荐运行方式

第一阶段建议采用：

- 手工点击运行
- 命令行运行
- 定时任务运行

三者选其一即可，不必一开始就做复杂前端界面。

---

## 10. 第一阶段的用户使用流程

建议使用流程如下：

### 步骤 1

物业助理或经理将当月资料放入指定目录。

### 步骤 2

人工触发 Agent 执行。

### 步骤 3

Agent 自动完成：

- 扫描文件
- 识别类型
- 抽取内容
- 汇总数据
- 生成报告初稿

### 步骤 4

物业经理审核输出内容，重点检查：

- 财务金额
- 欠费名单
- 合同与执照到期日
- 重大事件描述
- 委员会待决事项

### 步骤 5

审核后导出正式版本并发出。

## 11. 第一阶段的技术构成建议

从技术角度看，第一阶段不应过度设计，建议由以下 5 层组成。

### 11.1 文件采集层

负责读取：

- Excel
- PDF
- 图片

### 11.2 内容抽取层

负责：

- OCR
- 表格解析
- 文本摘要
- 关键信息提取

### 11.3 结构化映射层

负责将抽取结果映射到统一字段。

### 11.4 模板生成层

负责将结构化结果填充到月报模板中。

### 11.5 审核输出层

负责输出：

- 结构化结果
- Markdown 报告
- 错误与缺失项提示

---

## 12. 第一阶段的成功标准

第一阶段不要用“是否全自动”来判断成功，而应使用以下标准。

### 成功标准 1：流程可跑通

至少 1 个试点物业能跑通：

- 原始资料 -> AI 抽取 -> 报告初稿

### 成功标准 2：输出可读可审

生成的报告初稿应具备：

- 章节完整
- 内容可读
- 重点事项可识别
- 缺失项有标注

### 成功标准 3：节省时间

物业经理在报告准备上的时间明显下降。

### 成功标准 4：关键数据可追溯

至少关键数字和关键事项能追溯到来源文件。

---

## 13. 第一阶段推荐试点方式

建议采用“一个物业、一个月份、三类资料”的试点方式。

### 试点范围

- 物业数量：1 个
- 月份：1 个月
- 资料类型：Excel + PDF + 图片

### 试点输出

- 一份结构化抽取结果
- 一份 AI 自动生成的 Markdown 月报
- 一份人工修订后的最终版
- 一份差异分析总结

---

## 14. 第一阶段建议的角色分工

### 业务负责人

负责：

- 确认模板是否符合管理需要
- 确认报告章节和字段
- 审核输出结果

### 物业经理/管理处

负责：

- 提供原始资料
- 审核 AI 初稿
- 提供反馈

### 技术实施方

负责：

- 搭建 Agent 工作流
- 配置抽取逻辑
- 调整模板生成逻辑
- 修复提取错误

### AI Agent

负责：

- 抽取
- 汇总
- 草稿生成
- 缺失项提示

---

## 15. 对第一阶段的最终定义

如果需要一句话定义第一阶段，可以表述为：

**第一阶段交付的是一个用于自动生成物业管理报告初稿的 AI Agent V1，以及支撑其运行的模板、字段、归档和审核机制。**

换句话说：

- 它是一个 AI Agent 项目
- 但交付物不只是 Agent 本身
- 它是一个“可运行的小型 Agent 工作流系统”

---

## 16. 建议的下一步

在本定义明确后，下一步最应该补齐的是以下两份文档：

1. 《物业报告字段字典》
2. 《原始资料归档规范》

原因是：

- 字段字典决定 Agent 抽什么
- 归档规范决定 Agent 去哪里读

这两项确定后，第一阶段的 Agent V1 就可以开始真正落地实现。
